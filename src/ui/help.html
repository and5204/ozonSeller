Инструкция по работе с системой

Система предназначена для автоматического создания поставок через API с подбором доступных таймслотов. Поддерживаются два типа поставок:

  • Кроссдок (Crossdock) — поставка через точку отгрузки
  • Прямая поставка (Direct) — поставка напрямую на склад
---

1. Общий принцип работы

После создания заявки система:

  • Создаёт draft (черновик)
  • Ожидает его расчёта
  • Ищет доступные таймслоты
  • Выбирает самый ранний подходящий
  • Пытается создать supply (поставку)

Если таймслот исчезает или возникает ошибка — процесс автоматически повторяется.

---

2. Поля формы

Регион

Выбор зоны (например: RUS, SNG). Определяет доступные кластеры.

Кластер

Логистический кластер, в который будет создаваться поставка.

Начальное и конечное время

Формат: YYYY-MM-DDTHH:MM:SS

Пример:

2026-04-28T12:00:00

Важно:

  • Период не может превышать 28 дней
  • Если начальное время уже прошло — оно автоматически сдвигается
---

3. Кроссдок (Crossdock)

Дополнительные поля:

  • Поиск точки — поиск склада по названию
  • Точка отгрузки — склад, с которого отправляется товар

Система проверяет доступность таймслотов именно для выбранной точки.

Особенности:

  • Если нет таймслотов — система ждёт и пробует снова
  • Если точка невалидна — пересоздаётся draft
---

4. Прямая поставка (Direct)

Дополнительные поля:

  • Склад отгрузки — выбирается из кластера

Доступны только склады типа FULL_FILLMENT.

---

5. Товар и количество

  • SKU — выбирается из списка
  • Количество — целое число > 0
---

6. Статусы заявок

in_progressзаявка в работе
doneуспешно создана поставка
errorошибка (см. описание)
canceledостановлена пользователем

---

8. Особенности работы системы

  • Автоматические ретраи при rate limit (429, code=8)
  • Ожидание появления таймслотов (~9 минут)
  • Повторное создание draft при ошибках
  • Выбор самого раннего таймслота
---

10. Рекомендации

  • Ставьте широкий диапазон времени
  • Используйте корректные кластеры и склады
  • Не создавайте много заявок одновременно (rate limit)
